## 目录树

```
/rootproject/                 # 项目根目录 (Root Project)
│
├── README.md                 # 项目总览
├── PROJECT_STRUCTURE.md      # 本文件
├── INSTALL.md                # 安装指南
├── ROADMAP.md                # 开发路线图
│
├── OIPES root project/       # OIPES 生态服务（独立文件夹）
│
└── servecosys/             # ServEcosys 核心项目
    ├── base/                 # 底基系统源码与构建脚本
    │   ├── core/             # 底基系统核心（最小化运行时）
    │   ├── drivers/          # 底基系统可直接调用的硬件驱动（可选）
    │   └── tools/            # 恢复、刷写、调试工具
    ├── boot/                 # 引导启动程序链
    ├── kernel/               # 主内核文件夹
    │   ├── core/             # 内核中央
    │   └── modules/          # 内核模块 (pc/, mobile/, probe/, ...)
    ├── backend/              # 后端文件夹 (.smle)
    │   ├── sub_kernel/       # 后端子内核
    │   ├── security/         # 权限控制台与 SELinux
    │   └── oipes/            # 公共生态服务
    ├── frontend/             # 前端子内核文件夹 (.ssle)
    │   ├── ui/               # 界面层
    │   └── apps/             # 原生第三方应用
    ├── restore/              # 恢复文件区 (factory_image)
    └── scripts/              # 构建与快照管理脚本
```

## 权限阶梯详解

| 级别 | 名称 | 说明 | 控制权 |
|------|------|------|--------|
| 0 | r / rw / rx | 系统 |
| 1 | SANDBOX | 应用沙盒 | 系统 |
| 2 | USER | 普通用户/系统应用 | 系统 |
| 3 | DEBUG | 进阶调试 | 系统 |
| 4 | BL_UNLOCK | BL 解锁/特权文件 | 用户 |
| 5 | ROOT_SPLIT | Root 分能力/自定义恢复 | 用户 |
| 6 | MODULE_ROOT | 模块加载 Root | 系统 |
| 7 | KERNEL_ROOT | 内核加载 Root | 系统 |
| **8** | **SELINUX** | **SELinux 控制** | **用户** ⭐ |
| 9 | KMOD_LOAD | 内核模块加载 | 系统 |
| 10 | CUSTOM_KERNEL | 自定义内核 | 用户 |
| **11** | **BOOTLOADER** | **引导加载程序/启动链** | **用户** ⭐ |

### ⭐ 用户主权原则

**8 级和 11 级由用户完全控制：**

- **8 级（SELinux 控制）**
  - 用户可以自签 SELinux 策略
  - 不需要项目维护者许可
  - 不需要独立审计方批准

- **11 级（引导加载程序/启动链）**
  - 用户可以自签引导加载程序
  - 可以在自己的硬件上运行任何自己信任的代码
  - 完全的控制权

**核心理念：**
> 你的设备，你做主。

这是**以用户体验需求为中心，为用户服务**的根本体现。

## 架构设计的用户主权

### 1. 底基系统 - 用户的终极控制权

```
底基系统 (Base System)
- UEFI 层运行时
- 直接硬件访问
- 系统恢复功能
- 用户可编程
```

**意义：** 即使操作系统完全损坏，用户仍然可以通过底基系统恢复设备，不需要任何人的许可。

### 2. 双域隔离 - 用户的安全保障

```
后端安全域 (SED) - sys_dom_t
- SELinux 策略执行
- 权限裁决
- 安全决策

前端交互域 (UID) - uid_dom_t
- 用户界面
- 应用沙箱
- 交互处理
```

**意义：** 安全策略集中执行，用户清楚知道谁在访问什么，防止恶意软件窃取数据。

### 3. 权限阶梯 - 用户的选择权

```
0-3 级：日常使用
4-5 级：用户自由（BL 解锁、自定义恢复）
6-7 级：系统管理
8 级：用户主权（SELinux 控制）⭐
9-10 级：高级用户
11 级：用户主权（启动链控制）⭐
```

**意义：** 用户可以选择留在官方生态，也可以选择完全自治。**两者都是自由。**

### 4. OIPES - 用户的生态选择权

```
OIPES (开放生态服务)
- 服务端：中立非营利组织运营
- 客户端：完全开源
- 不捆绑特定云服务商
```

**意义：** 用户可以选择官方服务，也可以选择第三方兼容服务，甚至可以自建服务。

## 快速开始

```bash
# 1. 安装工具链
sudo apt install -y build-essential gcc make gnu-efi qemu-system-x86

# 2. 进入项目
cd rootproject

# 3. 编译
./scripts/build_kernel.sh full

# 4. QEMU 测试
./scripts/qemu_test.sh -d
```
