# ServEcosys Root Project

> 自由开源，为用户而生

ServEcosys 是一个基于 Linux 构建、面向 PC、移动端及多种设备形态的开源根项目（Root Project）。它的定位类似于 AOSP（Android Open Source Project），旨在提供一个自由、开源、集百家之长的通用操作系统底座。

## 核心纲领

继承 Linux 的自由与开放精神，以用户体验需求为中心，为用户服务。系统的一切设计——从架构到权限、从生态到恢复——都以用户的**掌控权**、**选择权**和**安全感**为出发点。

## 核心架构

```
│                    底基系统 (Base System)                    │
│                    UEFI 层 / 恢复环境                         │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                    唯一 Linux 内核                            │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────────┐  │
│  │  内核中央    │  │  设备模块集  │  │  SELinux 安全框架    │  │
│  │  (core)     │  │  (modules)  │  │  (不可降级)          │  │
│  └─────────────┘  └─────────────┘  └─────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
                              │
              ┌───────────────┴───────────────┐
              ▼                               ▼
┌─────────────────────────┐     ┌─────────────────────────┐
│   后端安全域 (SED)       │     │   前端交互域 (UID)       │
│   sys_dom_t             │     │   uid_dom_t             │
│   - SELinux 策略执行     │     │   - 显示服务             │
│   - 硬件抽象            │     │   - 应用沙箱             │
│   - 权限裁决            │     │   - 系统界面             │
│   - OIPES 客户端        │     │   - 输入管理             │
└─────────────────────────┘     └─────────────────────────┘
```

## 项目目录结构

```
/rootproject/                 # 项目根目录 (Root Project)
├── base/                     # 底基系统源码与构建脚本
│   ├── core/                 # 底基系统核心（最小化运行时）
│   ├── drivers/              # 底基系统可直接调用的硬件驱动（可选）
│   └── tools/                # 恢复、刷写、调试工具
├── boot/                     # 引导启动程序链
├── kernel/                   # 主内核文件夹
│   ├── core/                 # 内核中央
│   └── modules/              # 内核模块 (pc/, mobile/, probe/, ...)
├── backend/                  # 后端文件夹 (.smle)
│   ├── sub_kernel/           # 后端子内核
│   ├── security/             # 权限控制台与 SELinux
│   └── oipes/                # 公共生态服务
├── frontend/                 # 前端子内核文件夹 (.ssle)
│   ├── ui/                   # 界面层
│   └── apps/                 # 原生第三方应用
├── restore/                  # 恢复文件区 (factory_image)
└── scripts/                  # 构建与快照管理脚本
```

## 权限体系 (0-11 级)

| 级别 | 权限说明 |
|------|----------|
| 0 | 只读/只写/只执行 |
| 1 | 应用沙盒 |
| 2 | 普通用户/系统应用 |
| 3 | 进阶调试 |
| 4 | BL 解锁/特权文件 |
| 5 | Root 分能力/自定义恢复 |
| 6 | 模块加载 Root |
| 7 | 内核加载 Root |
| 8 | SELinux 控制 |
| 9 | 内核模块加载 |
| 10 | 自定义内核 |
| 11 | 引导加载程序/启动链 |

**关键设计原则：**
- **4 级 BL 解锁**是一道自由之门：解锁后，用户可以自签 SED 策略、自编内核
- **8 级和 11 级**的签名密钥由项目维护者及独立安全审计方**共管**，单方无法操作
- 用户通过**底基系统**拥有对设备的完整物理控制权
