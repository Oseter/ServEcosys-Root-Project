# ServEcosys Root Project

> 自由开源，为用户而生

## 根本纲领

**以用户主权为设计原则，以用户体验需求为落地基准，为用户服务。**

这是 ServEcosys 一切设计和决策的出发点和落脚点。

---

ServEcosys 是一个基于 Linux 构建、面向 PC、移动端及多种设备形态的开源根项目（Root Project）。它的定位类似于 AOSP（Android Open Source Project），旨在提供一个自由、开源、集百家之长的通用操作系统底座。其设计理念继承 Linux 的自由与开放精神，反厂商垄断锁定，技术机制保障「你的设备你做主、你的数据你掌控、你的生态你选择」。

## 核心纲领

继承 Linux 的自由与开放精神，**以用户主权为设计原则，以用户体验需求为落地基准，为用户服务**。系统的一切设计——从架构到权限、从生态到恢复——都以用户的**掌控权**、**选择权**和**安全感**为出发点。

## 项目目录结构

```

├── OIPES root project/       # OIPES 生态服务（独立文件夹）

    ├── base/                 # 底基系统源码与构建脚本
    │   ├── core/             # 底基系统核心（最小化运行时）

    │   └── tools/            # 恢复、刷写、调试工具
    ├── boot/                 # 引导启动程序链
    ├── kernel/               # 主内核文件夹
    │   ├── core/             # 内核中央
    │   └── modules/          # 内核模块 (pc/, mobile/, probe/, ...)
    ├── backend/              # 后端文件夹 (.smle)
    │   ├── sub_kernel/       # 后端子内核
    │   ├── security/         # 权限控制台与 SELinux
    │   └── oipes/            # 公共生态服务客户端
    ├── frontend/             # 前端子内核文件夹 (.ssle)
    │   ├── ui/               # 界面层
    │   └── apps/             # 原生第三方应用
    ├── restore/              # 恢复文件区 (factory_image)
    └── scripts/              # 构建与快照管理脚本
```

## 权限体系 (0-11 级)

| 级别 | 权限说明 | 控制权 |
|------|----------|--------|
| 0 | r / rw / rx | 系统 |
| 1 | 应用沙盒 | 系统 |
| 2 | 普通用户/系统应用 | 系统 |
| 3 | 进阶调试 | 系统 |
| 4 | BL 解锁/特权文件 | 用户 |
| 5 | Root 分能力/自定义恢复 | 用户 |
| 6 | 模块加载 Root | 系统 |
| 7 | 内核加载 Root | 系统 |
| **8** | **SELinux 控制** | **用户** ⭐ |
| 9 | 内核模块加载 | 系统 |
| 10 | 自定义内核 | 用户 |
| **11** | **引导加载程序/启动链** | **用户** ⭐ |

**关键设计原则：**

- **8 级和 11 级由用户完全控制** ⭐
  - 用户可以自签 SELinux 策略
  - 用户可以自签引导加载程序
  - 不需要任何人的许可

- **4 级 BL 解锁是一道自由之门**
  - 解锁后，用户可以在自己的硬件上运行任何自己信任的代码
  - 这是"为用户服务"的终极体现——你的设备，你做主

- **用户自签链上无法连接 OIPES 高等级服务**
  - 这不是惩罚，而是技术必然
  - 因为自签链的信任根不在官方信任锚点之内
  - 用户可以选择留在官方生态享受无缝服务，也可以选择完全自治
  - **两者都是自由**

## 快速开始

### 1. 安装编译工具链

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install -y build-essential gcc make binutils bc kmod cpio \
    libncurses-dev bison flex libssl-dev libelf-dev libfdt-dev \
    device-tree-compiler openssl git wget qemu-system-x86 gnu-efi
```

### 2. 克隆项目

```bash
git clone https://github.com/Oseter/ServEcosys-Root-Project.git
cd ServEcosys-Root-Project
```

### 3. 编译

```bash
# 查看构建选项
ls -la scripts/

# 执行构建（具体参考 scripts/README.md）
./scripts/build.sh
```

## 系统自愈

ServEcosys 使用 Btrfs 文件系统配合多子卷布局：

- `@system` - 系统只读基准子卷
- `@data` - 用户数据子卷（独立）
- `@snapshots` - 自动快照存储子卷

**恢复机制：**
1. **轻症** - 引导程序选择上一个健康快照，10 秒内恢复
2. **重症** - 从 `/dev/recovery` 分区重装，保留用户数据
3. **终极恢复** - 从底基系统（UEFI 层）手动修复

## OIPES 开放生态

OIPES（操作系统互联网公共生态服务）提供与生态系统的标准化连接：

- **服务端** - 由中立非营利组织独立运营
- **客户端** - 运行于后端安全域（backend/oipes/）
- **不捆绑**任何特定云服务供应商
